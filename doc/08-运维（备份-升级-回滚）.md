# 运维（备份 / 升级 / 回滚）

> [< 上一篇：安全基线](07-安全基线（强制项）.md) | [返回目录](README.md) | [下一篇：故障排查 >](09-故障排查.md)

## 备份（必须）

- PostgreSQL：每日全量（至少）+ 异机保存
- 配置：`/opt/litellm-server/config/`、反代配置、`.env`（注意权限）

## 升级（必须先预生产）

- 先在预生产验证：接口、指标、WordPress 集成兼容
- 生产升级前做 DB 备份
- 明确回滚策略：镜像版本回退 + DB 恢复预案

## 回滚（建议演练）

- 回退 LiteLLM 镜像版本到上一稳定版本
- 恢复 PostgreSQL 备份到升级前快照
- 验证 `/health`、核心 `/v1/...`、Key 管理、指标采集
